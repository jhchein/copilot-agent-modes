# Scenario: New Adopter

## Role

- Developer evaluating this template for first-time use in an existing repository.

## Decision concerns

- Can I understand the structure quickly and start safely with minimal confusion?
- Are mode boundaries and handoffs clear enough to avoid accidental over-scoping?
- Does `/bootstrap` clearly separate baseline setup from optional patterns?

## Risk tolerance

- Low tolerance for setup complexity.
- Moderate tolerance for optional advanced patterns if they are clearly off by default.

## Review focus

- Time-to-value from README and prompt cheat sheets.
- Clarity of "what is default" vs "what is optional".
- Consistency between examples and what `/bootstrap` proposes.

## Scoring dimensions

Score each dimension 0-10 with concrete evidence.

1. **Onboarding clarity**: A new user can understand structure and first steps quickly.
2. **Default-vs-optional clarity**: Baseline setup and optional patterns are clearly separated.
3. **Workflow usability**: Mode boundaries and handoffs reduce confusion.
4. **Risk transparency**: Trade-offs and open questions are explicit.
5. **Writing quality**: Clear, precise language aligned to `.github/instructions/writing.instructions.md` when available.
