# Writing Quality Skill

Use this when editing or generating user-facing documentation under `docs/`.

## Process

1. **Orient**
   - Read the full target document.
   - Identify the audience, purpose, and required technical truth.

2. **Structural pass**
   - Ensure each section has a single clear purpose.
   - Split overloaded paragraphs and remove redundant headers.

3. **Language pass**
   - Remove filler and repetitive transition phrases.
   - Replace vague wording with concrete technical statements.

4. **Accuracy pass**
   - Mark assumptions and unknowns explicitly.
   - Keep claims aligned with source material and decisions.

## Output contract

- Summarize key quality issues found.
- List meaningful rewrites performed.
- State any limitations or unresolved ambiguities.
